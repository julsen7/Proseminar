In chapter 1 we will explore what Logo is, its main features and how to work with it. We will show you some base commands which are required to better understand it in other chapters. In chapter 2, we will methodically explore the concepts of recursion and fractals using the Pythagoras tree, Koch's flake. \subsection{Definition of Logo and its origin}
\label{sec:definitionlog}
Before we move on to recursion and fractals, we're going to find out what Logo is and learn some base commands.

Logo is a programming language that is well-known for its use of turtle graphics. With Logo, you can draw different figure shapes.

It was invented in 1966 in Cambridge (USA) by Wally Feurzeug, Seymour Papert, Daniel Bobrow and Cynthia Solomon.

The original intention behind the creation of this language was to provide a means for children in schools to learn how to code and enhance their mathematical skills.

\subsection{Turtle Graphics}
\label{sec:turtleg}
The turtle is a pen or a cursor (often represented as a triangle) that moves after a command and draws when the pen is down. You can change the position, direction, pen state, color etc. It will all be displayed on the computer screen.

\subsection{Data Types}
\label{sec:datatypes}
In Logo, data types are dynamic. This means that they do not need to be defined explicitly.

There are four main data types: word, number, list and boolean.
\begin{enumerate}
  \item \textit{word} is used for a sequence of characters
  \item \textit{number} for such numbers as integer, float, double etc.
  \item \textit{list}  is a list of numbers or characters (which may for example be a sentence or an array).
  \item \textit{bool} is for true or false
\end{enumerate}

\section{Recursion}
\label{sec:recursion}
We are now capable of writing programs with these base commands but we still have to introduce recursion for a functional programming language like Logo.

\subsection{Concept}
Recursion has its origin in the latin word \textit{recurrere (to go back)}. A recursive written function has to contain three key parts:

\begin{enumerate}
  \item Base case
  \item Calling itself
  \item Recursive step
\end{enumerate}

The base case will be a condition that indicates on what case our recursive calls will end up terminating.
We need this due to our limitations of storage and to avoid an infinite amount of calls on our call stack.
To acctually reach this base case we need a recursive step that needs to be provided by our recursive call that approaches our base case \cite{geeksforgeeks}.

\begin{lstlisting}[title={Fibonacci numbers}, language=Java, label={lst:fibonacci}]
  private static int fib(int n) {
    if (n <= 0) return -1; // wrong input
    if (n == 1) return 0; // base case 1
    if (n == 2) return 1; // base case 2
    return fib(n - 1) + fib(n - 2); // recursive call
    }
\end{lstlisting}

In this java implementation of the Fibonacci numbers (\ref{lst:fibonacci}) you can see that there are also multiple base cases possible.
For this example the first Fibonacci number is defined to be 0 and the second one has to be 1. After these base cases the recursive call and its embedded recursive steps will gie us our final result of our requested nth number.
As you might have noticed the recursive step is not only one times but two times in this example. This is due to the fact that the Fibonacci numbers are defined as the sum of the two previous numbers.

Very important is that the recursive step actually reaches our base case. In this example each reduction (\(n-1\) and \(n-2\)) will either reach our first or our second base case and terminate.
If we would have skipped our base case and had not filtered out wrong inputs by \(n<=0\) the method would not terminate and the recursion will end up in an infinite loop.

The phrase \textit{depth of  recursion} determines the amount of nested recursive funtions inside of itself.

Recursion often finds usage in science like mathematics \textit{eg. factorials, Fibonacci} or computer science like in functional programming languages like Logo that profit from reduced lines of code and readability. 

\subsection{Example: Pythagoras tree}

\begin{figure}[t]
  \centering
  \includegraphics[width=0.5\textwidth]{images/pythagoras_tree.png}
  \caption{Pythagoras tree}
  \label{fig:Pythagoras tree}
\end{figure}

The pythrgoras tree is a very common example for explaining recursion \cite{mathworld}.
Its main shape is a square with recursively placed squares on top of the base square rotated by a predefined angular that calculates based on the famous Pythagoras formula (\ref{fig:Pythagoras tree}).

To implement this we will use the previously explained scheme: We declare our procedure within the first line (keyword \textit{to}) with its name being \textit{pythagoras} an two needed parameters (indicated by the colons) as \textit{length} and \textit{depth} and end our definition with the keyword \textit{end} in our last line.

\begin{lstlisting}
  to pythagoras :length :depth
    ...
  end
\end{lstlisting}

First we implement our main shape (the square) by repeating four times going forward and turning right.

\begin{lstlisting}
  repeat 4 [fd :length rt 90]
\end{lstlisting}

Now we write down our base case which means to terminate our programm at a reached depth of 0. The depth represents the maximum amount of squares placed on top of each other (recursion depth).

\begin{lstlisting}
  if :depth = 0 [stop]
\end{lstlisting}

We and up at our starting position at the left bottom corner of the just drawn sqaure. To draw our recursive sqaures we need to correct our position.

For that we have to reach the top left corner and rotating to the left by 45 degree which will be our predefined rotation so both squares will have the exact same length and our Pyrthagoras tree becomes axially symmetric.

\begin{lstlisting}
  fd :length lt 45
\end{lstlisting}

Our recursive squares are then drawn with a reduced length in the scale of the Pythagoras formula and a depth that decreases by 1 which ensures the base case of depth 0 will be reached at some point.

\begin{lstlisting}
  pythagoras :length / (sqrt 2) :depth - 1
\end{lstlisting}

Because we are using an angle of 45 degree the following derivation explaines the new length:
We assume our hypothenusis as \textit{a} and our two kathets as length \textit{b}.

According to Pythagoras:

\begin{align*}
  a^2 + a^2 &= b^2 \\
  2a^2 &= b^2 \\
  \sqrt{2}a &= b \\
  b &= \frac{a}{\sqrt{2}}
\end{align*}

We have to correct our position again to start the second recursive square.

\begin{lstlisting}
  rt 90 fd :length / (sqrt 2)
\end{lstlisting}

Positioned correctly, we perform the same recursiove call as our first part.

\begin{lstlisting}
  pythagoras :length / (sqrt 2) :depth - 1
\end{lstlisting}

After each recursive depth we return to the position of the previous one (\textit{depth + 1}).

\begin{lstlisting}
  bk :length / (sqrt 2) lt 45 bk :length
\end{lstlisting}

Our final procedure will result in the following:

\begin{lstlisting}[title={Pythagoras tree}]
to pythagoras :length :depth ; variable length and depth
  repeat 4 [fd :length rt 90] ; square
  if :depth = 0 [stop] ; base case
  fd :length lt 45 ; change pos
  pythagoras :length / (sqrt 2) :depth - 1 ; recursive left
  rt 90 fd :length / (sqrt 2) ; change pos
  pythagoras :length / (sqrt 2) :depth - 1 ; recursive right
  bk :length / (sqrt 2) lt 45 bk :length ; go back
end
\end{lstlisting}

\section{Fractals}

\subsection{Concept}

To understand the possibilities of graphical approximation Logo has we need to introduce the concept of fractals.
Fractals are the idea of performing a recursion with a \textit{recursion depth} of infinity and are selfsimilar which means they are made up of smaller copys of it self. 
You can imagine them by "breaking" (\textit{lat. fracere}) them apart without being able to differentiate them from its parts \cite{alfonseca1996representation}.

Fractals are a theoretical concept that cannot be used in the sense of what they mean due to physical limitations of the real world like the finite resolution a monitor has. Some well-known fractals from nature are the golden ratio (\ref{fig:Golden ratio}) or the Sierpiński triangle (\ref{fig:Sierpiński triangle}) which was long considered a purely mathematical concept.
However, this form also occurs in the form of the enzyme citrate synthase \cite{maxplanckgesellschaft}.

\subsection{Example: Koch flake}

\begin{figure}[t]
  \centering
  \includegraphics[width=0.4\textwidth]{images/koch_flake.png}
  \caption{Koch flake}
  \label{fig:Koch flake}
\end{figure}

The Koch flake became quite popular for its simplistic form and minimal lines of code for a quite nice looking shape of a snowflake.
First des cribed by Helge von Koch in 1904 is is made up of three triangular shaped Koch curves which follow simple rules \cite{mathworld2}.

At first a Koch curve simply is a line with a predefined length. By increasing the depth, there will be placed Koch flakes on top of these linees forming a tringular shaped spike on top of the base line. 
A depth of 0 means just a straight line and the higher the depth the more triangles are placed on top of each other.

We will declare our Koch curve in the same way as we did with the Pyrthagoras tree.

\begin{lstlisting}
  to curve :length :depth
    ...
  end
\end{lstlisting}

We will start by defining our terminiation condition (\textit{base case}) by drawing a straight line.

\begin{lstlisting}[title=Part 1/4:]
  ; base case: if depth is 0, move forward and stop
  if :depth = 0 [fd :length stop]
\end{lstlisting}

For achieving our Koch curve with a triangular shape placing, our Koch curve consists of four parts: 

The first quarter section features a straight stretch spanning one-third of the length. The second and third quarter represent the rotation and both of the triangle sides placed a depth on top. To finalize the curve we end up performing the same third of the length drawn in step one.

\begin{lstlisting}
  ; fractal part 1/4 (move)
  curve :length / 3 :depth - 1
  ; fractal part 2/4 (turn left and move)
  lt 60
  curve :length / 3 :depth - 1
  ; fractal part 3/4 (turn right and move)
  rt 120
  curve :length / 3 :depth - 1
  ; fractal part 4/4 (turn left and move)
  lt 60
  curve :length / 3 :depth - 1
\end{lstlisting}

To get a snowflake-shaped Koch flake we simply need to expend the Koch curve by another small procedure \textit{flake}.
This flake repeats our \textit{curve} three times and rotates by 60 degree between each curve so we end up having the shape of a triangle (\ref{fig:Koch flake}).

\begin{lstlisting}[title={Kock flake}]
  ; procedure with parameters length and depth
  to curve :length :depth
    ; base case: if depth is 0, move forward and stop
    if :depth = 0 [fd :length stop]
    ; fractal part 1/4 (move)
    curve :length / 3 :depth - 1
    ; fractal part 2/4 (turn left and move)
    lt 60
    curve :length / 3 :depth - 1
    ; fractal part 3/4 (turn right and move)
    rt 120
    curve :length / 3 :depth - 1
    ; fractal part 4/4 (turn left and move)
    lt 60
    curve :length / 3 :depth - 1
  end

  to flake :depth
    ; 3 sides of base triagle form
    repeat 3 [
      curve 200 :depth
      rt 120 ; turn right to next triangle base side
    ]
  end
\end{lstlisting}

\section{Lindenmayer-System}
The Lindenmayer-System or short the L-system was invented by Aristid Lindenmayer in 1968. He was a Hungarian Biologist who wanted to recreate plant growth as accurate as possible and for this purpose he came up with this $(V, \omega, P)$. V is the Alphabet which consists of non-terminal symbols and terminal symbols. Non-terminal symbols are symbols that will be replaced eventually, whereas terminal symbols will not be replaced at all. $\omega$ is a non-empty starting word of an L-system which will be used to form new words that will be part of the L-system. These words are created with Production rules P.
\subsection{rewriting methods}
This paper will only discuss the deterministic and context-free L-system. We start with an example to get the basic concept of those systems . When looking at $(\{a,\ b\},\ a,\ P)$ where the production rule P describes $a \rightarrow ab$ which means whenever a word contains the letter $a$ it is replaced in the next step for $ab$ and $b \rightarrow bb$ which means every $b$ will be replaced for $bb$. We start with the letter a because our starting word $\omega =a$ in our system $a$ is replaced for $ab$. now we have two letters in our word and each of them will be replaced and they are replaced simultaneously. This is a big difference compared to normal context-free languages where only one symbol is replaced at a time. $ab$ becomes $abbb$ because the $a$ becomes $ab$ and the $b$ turns to $bb$, next $abbb$ turns to $abbbbbbb$ (Figure \ref{bsp_productionrules}).

We will discuss two ways to create an L-System using the turtle-graphics. They are edge-rewriting and node-rewriting. In these rewriting systems the alphabet $V=\{+,-,F\}$. These characters form words that can be interpreted in the turtle graphics. $+$ means turning the head of the turtle to the right and $-$ means turning to the left, both with a predefined angle that does not change in one system. and $F$ stands for going forward.
The production rules in edge-rewriting replace the $F$, the edges for a new shape. The edges in node-rewriting do not get replaced, there are new symbols $A\notin F$ that have no interpretation in the turtle-graphic. Their only purpose is to be replaced.

\subsubsection{Edge rewriting}
Edge-rewriting is the more intuitive way of creating L-systems. For example, Figure \ref{edge_bsp} shows an L-system $(\{+,-,F\},F,P)$ with $P:=F \rightarrow +F-F$ and an angle $\delta=45$°. As shown earlier, L-systems have terminal symbols and non-terminal symbols. In our example $\{+,-\}$ are terminal symbols, they do not get replaced by our production rule and $F$ is a non-terminal symbol since we have a rule that replaces the $F$ for something else. However our $F$ already had a meaning, namely being an edge in our L-system and so our edge is replaced for a shape as you can see in Figure \ref{edge_bsp}. There is also the possibility to use two $F$, for example having now $F_r$ and $F_l$ for more variation and complexity in our L-system. So now we define $P:= F_r\rightarrow +F_r-F_l ,\ F_l\rightarrow -F_r+F_l$ and we start with the starting word $\omega=F_r$. The first rule is the same rule as in the earlier example and our second rule is the same shape to replace our lines but mirrored to the first rule. As seen in Figure \ref{edge_bsp}, with two reproduction rules, the figure looks more complex.
\begin{figure}[t]
  \centering
  \includegraphics[width=0.5\linewidth]{images/edge_bsp.png}
  \caption{Example of L-system using edge-rewriting. 1. is before any rule was applied, 2. has a depth of 1. and 3.1 and 3.2 each have a depth of 2.  3.1 was created with one production rule, 3.2 was created with 2 production rules.}
  \label{edge_bsp}
\end{figure}
\subsubsection{Node rewriting}
For Node-rewriting we do not replace our edges $F$ anymore. We introduce new symbols $\{A,B,C,...\}$ that do not have any meaning in our system already. We will only use the symbol $A$ in our example (Figure \ref{node_bsp}). Our new L-system $(\{+,-,F,A\},FA,P)$ with $P:=A\rightarrow+AF-AF$ and the angle $\delta=45$°. Since the non-terminal symbols do not have any meaning, they do not change the image that we get by illustrating our L-system, but they determine how the next step of the recursion will look like. As you can see in Figure \ref{node_bsp}, the edges that have been drawn in those with less depth, they are also in those with higher depth with new shapes in between them. Again we can create L-systems with multiple rules. We define $P:= A_r\rightarrow+A_r F -A_l F,\ A_l\rightarrow-A_rF+A_lF$ and we choose $\omega=FA_r$. Just like with edge-rewriting the first two recursion steps look similar to the L-system with only one rule but with further depth the difference is clear.

\subsection{Designs}
The next step for making our fractals look more fabulous is to use designs. Designs are basically an ending repetition of our fractal. Just like we have seen earlier with the koch flake that we have created by repeating the koch curve three times. The first step is to choose any polygon as base for the rewriting. In our example (Figure \ref{muster_bsp}) we will choose a square. With our square, we now have four edges already as base whereas we had only one in our fractal, for edge-rewriting instead of replacing our only line, we now have four lines, each being replaced by our shape (Figure \ref{muster_bsp} left side). We can do the same for node rewriting by placing nodes on the edges that every corner has a node and we replace those nodes for our shape. every time we replace our edges or nodes in any polygon, the line stops there, where it had begun (Figure \ref{muster_bsp})

\subsection{Branching}
Using edge/node- rewriting, we can now create many L-systems. By using the turtle graphics, we can draw any number of angles and lines like the Pythagoras tree, the Koch flake or any self made L-system lines or designs. But they will always just be a line, however the world of plants is nothing but a world of branching. Lindenmayer was a Biologist and he wanted to recreate the world of plants. This too is possible to be made both with edge- and node-rewriting but we will only discuss it with edge-rewriting, but it works the same way with node-rewriting.
\subsubsection{Stack}
First we need to know what a stack is and how it works for us to use branching. A stack is a data structure to safe data. Two functions are used to operate the stack. The first function is push(input). When calling it, input will be written in our stack. The second function is pop(). It does not get any parameters and returns the last thing that was written on the stack removing it at the same time from it. So the stack uses the principle of Last in First out or "LiFo".
\subsubsection{edge-rewriting with stack}
We will introduce two new symbols in our alphabet. We define $[:= push((position,direction))$ and $]:= pop()$. Every time that we will use $[$ in a word of our system we save the current position and the direction the turtle is facing and every time we read $]$ in our word the turtle returns to that position and the direction faced when we pushed that position. In our Example we used the L-system $(\{-,+,F,[,]\},F,P)$ with $P:=F\rightarrow F[-F][+F]F$ and $\delta=45$°. In Figure \ref{branch_bsp} we can see how we already have three branches with a depth of 1 in the 2. image. At the 3. image we can see why the stack is important, and why we could not use a list with "FiFo". The word for our 3. image is $F[-F][+F]F[-F[-F][+F]F][+F[-F][+F]F]F[-F][+F]F$ and when we interleave the braces like we did in that word we do not wanna mix up the order, because if we did, the image would already consist of three parts in our example which would not be a fractal anymore.
